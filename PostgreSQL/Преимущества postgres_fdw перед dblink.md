# Преимущества postgres_fdw перед dblink

Расширение [postgres_fdw](https://postgrespro.ru/docs/postgresql/current/postgres-fdw) позволяет эффективно взаимодействовать с внешними таблицами и представлениями (которые находятся в другом кластере БД) так же, как с локальными. К внешним таблицам можно выполнять `SELECT/INSERT/UPDATE/DELETE/TRUCATE` запросы.

Для вызова внешних хранимых функций (`SELECT func()`), процедур (`CALL proc()`) и выполнения других DML запросов необходимо использовать расширение [dblink](https://postgrespro.ru/docs/postgresql/current/dblink), но в качестве подключения использовать ранее созданное именованное соединение!

№|Критерий сравнения|[postgres_fdw](https://postgrespro.ru/docs/postgresql/current/postgres-fdw)|[dblink](https://postgrespro.ru/docs/postgresql/current/dblink)
-|------------------|------------|------
1|Возможность отображать внешние таблицы (с мета-данными) в списке таблиц БД в psql и в большинстве GUI клиентов-навигаторов PostgreSQL (все IDE от JetBrains, DBeaver, Visual Studio Code и др.)|да|нет
2|Возможность обращаться к внешним таблицам так же, как к обычным: читать и модифицировать (если есть соответствующие права доступа во внешней БД) без использования функций-прослоек типа dblink*().|да|нет
3|Возможность хранить и изменить настройки подключения (хост, порт, имя пользователя и пароль) для внешней таблицы, представления, хранимой функции в одном месте (в настройках подключения к внешнему серверу), без модификации мета-данных или исходного кода объекта БД.|да|нет
4|Возможность эффективного (быстрого) выполнения соединения (join) внешних таблиц с обычными таблицами, т.к. планировщик использует мета-данные внешних таблиц.|да|нет
5|Возможность эффективного (быстрого) выполнения триггеров, срабатывающих для каждой строки (FOR EACH ROW), т.к. вместо переподключения используются постоянное соединение к внешней БД.|да|нет
